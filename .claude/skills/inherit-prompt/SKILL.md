現在の会話・コードベース・git 状態をもとに、**新しい Claude Code セッションに
そのまま貼り付けて作業を再開できるプロンプト**を生成する。

出力は次のセッションの Claude への「最初のメッセージ」として機能する。
読み手は今のセッションを一切知らない前提で書く。

## 出力ルール（絶対厳守）

- チャットに直接出力する（ファイルに書き出さない）
- **出力全体を1つの fenced code block で囲む**（開始行: ``` のみ、終了行: ``` のみ）
  → UI の「Copy code」ボタン1クリックで全文コピーできる
- コードブロック内部でバッククォート3つを使わない（ネスト不可のため）
- コードブロック内部でのコマンドは 4 スペースインデントで表現する
- コードブロックの前後に説明・前置きを一切書かない（ブロックだけ出力する）

## 生成手順

1. git log --oneline -5 を実行してコミット履歴を確認する
2. 会話履歴から「判明した事実」「完了済み」「残タスク」を抽出する
3. 関連ファイルをファイルパス:行番号付きで列挙する
4. 下記テンプレートを **fenced code block で囲んで** 出力する

## テンプレート

（以下の内容を ``` ... ``` で囲んで出力すること）

あなたは [プロジェクト名] の開発を引き継ぐ Claude Code です。
以下のコンテキストを読んで、「残タスク」を上から順に実装してください。

## プロジェクト概要

<プロジェクトの目的・技術スタックを2〜3行で>

## 前セッションで判明した事実

- <ファイルパス:行番号> <何がわかったか>
- ...

## 完了済み

- <すでに実装・コミット済みの内容>

## 残タスク（上から順に実装）

1. <ファイルパス:行番号> を <具体的にどう変える>
2. ...

## 関連ファイル早見表

    パス                                        役割
    ──────────────────────────────────────────────────────────
    path/to/file.rs:123                         <一言>

## 実装上の注意

- <やってはいけないこと・破壊的変更になる可能性があること>

## 作業開始の手順

    git status
    git log --oneline -5
    <ビルド確認コマンド>

まず上記コマンドで現在の状態を確認してから実装を開始してください。
