# Miharashi を Torimi（鳥見）に改名する

status: accepted

Date: 2026-07-07

## Context

dev-client の名前 **Miharashi（見晴らし）** は、語感が牧歌的で Hayate（疾風）・Hayabusa（隼）・
Tsubame（燕）の疾走感あるラインナップから浮いている、という不満が出発点（grill セッション
2026-07-07）。改名候補として当初 **Chokan（鳥瞰）** が挙がったが、検討の過程で
「鳥*が*見る」ではなく「鳥*を*見る」道具である、という本質が確認された。

改名は次の 3 条件を満たすため ADR に記録する: 名前はコード・パッケージ名・CI・wire
グローバルに広く焼き付き戻しにくい／将来の読者が「なぜ Torimi?」と不思議に思う／
実際に複数候補のトレードオフから選んだ。

## Decision

**Torimi（鳥見）** に改名する。根拠は二重の意味:

1. バードウォッチャー用語の「鳥見」＝探鳥。**鳥（フレームワーク）の動きを見る道具**を
   文字通り言い当てる。
2. 江戸幕府の役職「**鳥見役**」— 鷹場（鷹狩りの狩場）を監視・管理した役人。
   「隼や燕が飛ぶ場を見張り、整える番人」として世界観との接続が現行の「高台」より強い。

和語（訓読み）3 モーラで Hayate / Tsubame / Hayabusa の音族に揃い、ローマ字 `torimi` は
短く一意で衝突もない。

適用範囲は**後方互換シム無しの全面リネーム**とする（ディレクトリ・`@torimi/*` パッケージ
スコープ・wire 契約グローバル・Rust/Kotlin 識別子・CI workflow・タグ規約・Worker 名）。
決定時点でリリースタグ・Play 配布・Worker デプロイはいずれも存在せず（GitHub releases ゼロ、
demo endpoint 未デプロイ）、外部互換制約が無いため protocol version の bump も不要。

## Considered Options

- **Chokan（鳥瞰）** — 却下。(i) 鳥瞰は「鳥が上から見る」視点で、鳥＝フレームワークという
  既存語彙体系では dev-client を鳥側（＝フレームワークの一種）と誤読させる。(ii) 唯一の
  漢語（音読み）で和語の音族から浮く。(iii) ローマ字 `chokan` が長官・朝刊と衝突し検索性が悪い。
- **Tanchō（探鳥）** — 却下。意味は正確だが漢語で、`tancho` が丹頂（鶴）と衝突して
  やはり「鳥側」に誤読される。
- **Monomi（物見）／Takami（高見・鷹見）** — 意味論は近いが、「鳥を見る」ことを直接
  言わない（物見は対象が汎用、高見は傍観の含意、鷹見は燕が対象外に聞こえる）。
- **Miharashi 維持** — 却下。景色を漫然と眺める語感が残り、不満の本質（鋭さの欠如）を
  解消しない。

## Consequences

- 既存 ADR（0001〜0003 ほか Miharashi に言及する各 context の ADR）は**本文を書き換えず**、
  ADR-0001 冒頭の Tsubame 改名時と同型の「用語更新」バナーを冒頭に足すのみ（決定当時の
  記録として原文を保つ、この repo の既存慣習）。
- 初回リリース（#744: タグ `miharashi-android-v1`・Play アップロード・workers.dev URL 確定）
  より**前**にリネームを済ませるのが望ましい。外部に旧名が焼き付いてからでは全面リネームの
  前提（互換制約ゼロ）が崩れる。
